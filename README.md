# IM_PROJEKT-PARKING
# Kurzbeschreibung
Der Parkinator ist ein interaktives Medienprojekt, das die durchschnittliche Parkplatzbelegung in St. Gallen aufzeigt. Nutzer:innen können Wochentag und Uhrzeit filtern und sich anschliessend die durchschnittliche Belegung verschiedener Parkplätze und Parkhäuser anzeigen lassen. Über eine Detailansicht werden zusätzlich der Standort via Google Maps sowie eine genauere Grafik zur Belegung über den ganzen Tag dargestellt.

# Learnings
•	Wir haben während des Prozesses gemerkt, dass bei dem Kuchen Diagramm eine Prozentangabe (wie im Figma) nicht sinnvoll ist. Deshalb haben wir nun die genaue Anzahl freier und belegter Parkplätze hinterlegt.

•	Bei einem weissen Hintergrund sollte man immer darauf achten, dass die Schrift auf schwarz (und nicht auf weiss) eingestellt ist. Sonst sucht man eine Stunde nach dem Fehler, obwohl das Gewollte schon dort war aber man hat es nicht gesehen :)

•	Ctrl + Shift + R (Hard Refreshing) kann viele Probleme lösen, wenn Änderungen nicht sichtbar sind.

•	Die Coachings im Unterricht haben extrem geholfen und viele Probleme gelöst.

# Schwierigkeiten
Die grösste Herausforderung war die Zusammenarbeit mit einer lokalen Datenbank. Da immer nur eine Person Zugriff auf die Daten hatte, musste das Projekt logisch aufgeteilt werden. Besonders das Styling der Datenausgabe war schwierig, da es nur von der Person mit Datenzugriff getestet werden konnte. Zusätzlich nahm das Responsive Design viel Zeit in Anspruch, bis es auf allen Geräten stimmig funktionierte.

# Benutzte Ressourcen
Genutzt wurden Visual Studio Code mit GitHub Copilot, ChatGPT sowie Coachings vor Ort. ChatGPT war hilfreich für einfache Anpassungen oder zum Aufräumen von Code, funktionierte bei komplexeren Problemen jedoch nur begrenzt. Dort mussten wir schrittweise vorgehen und sehr konkrete Prompts formulieren. Die Coachings vor Ort waren insgesamt am effektivsten und haben uns am meisten weitergebracht.
